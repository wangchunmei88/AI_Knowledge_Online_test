GPT 系列 (OpenAI)

OpenAI 2018 - 至今 Decoder-only

📌 一句话定位

GPT = Generative Pre-trained Transformer
纯解码器架构的自回归语言模型，通过「预测下一个词」学习语言，是当今最主流的大模型架构范式。

🏗️ 核心架构

# GPT 核心结构（伪代码） class GPTBlock: def forward(x): # 1. Masked Multi-Head Self-Attention x = x + Attention(LN(x)) # 只能看左边，不能偷看未来 # 2. Feed-Forward Network (FFN) x = x + FFN(LN(x)) # 通常用 GELU/SwiGLU return x
组件 GPT-2/3 GPT-4
架构 Decoder-only Decoder-only (推测)
位置编码 可学习位置嵌入 未公开（可能改进）
激活函数 GELU 未公开
归一化 LayerNorm Pre-Norm
上下文长度 2K (GPT-3) 128K

💡 核心特点

  1. Decoder-only：只有解码器，没有编码器，适合生成任务
  2. Causal Mask：自注意力带掩码，只能看到已生成的 token
  3. 自回归生成：逐 token 预测 P(x_t | x_<t)，从左到右生成
  4. 预训练任务：语言建模（Next Token Prediction）

🔑 关键技术演进

GPT-1 (2018)

首次提出「预训练 + 微调」范式，1.17亿参数，12层 Transformer Decoder

GPT-2 (2019)

15亿参数，Zero-shot 能力显现，发现大模型的涌现能力

GPT-3 (2020)

1750亿参数，In-context Learning，Few-shot 无需微调

GPT-4 (2023)

多模态（文本+图像），推理能力大幅提升，具体规模未公开

🎯 面试常考点

  • 为什么 GPT 用 Decoder-only 而不是 Encoder-Decoder？
    → 生成任务不需要编码器的双向注意力，Decoder-only 更简单高效，且适合自回归生成
  • GPT 的预训练任务是什么？
    → 语言建模（Language Modeling），最大化 P(x_t | x_<t) 的概率
  • Causal Mask 怎么实现？
    → 上三角矩阵 mask，将未来位置的 attention score 设为 -∞

LLaMA 系列 (Meta)

Meta AI 2023 - 至今 Decoder-only

📌 一句话定位

LLaMA = Large Language Model Meta AI
开源社区最具影响力的基础模型，用更小的参数规模达到 GPT-3 级别的效果，是当今开源大模型的「基座」。

🏗️ 核心架构改进

1️⃣ RoPE 旋转位置编码

LLaMA 使用 RoPE (Rotary Position Embedding) 替代绝对位置编码：

# RoPE 核心思想：通过旋转矩阵编码相对位置 # 对 Q, K 做旋转变换，使得 q_m^T · k_n 只依赖于 (m-n) q_rotated = rotate(q, position=m) # 位置 m 的 query k_rotated = rotate(k, position=n) # 位置 n 的 key attention_score = q_rotated^T · k_rotated # 自动包含相对位置信息

优势：更好的长度外推性，可以处理比训练时更长的序列。

2️⃣ SwiGLU 激活函数

FFN 层使用 SwiGLU 替代 ReLU/GELU：

# SwiGLU: 门控机制 + Swish 激活 FFN(x) = (x · W1 ⊗ Swish(x · W2)) · W3 # 对比传统 FFN: # ReLU: FFN(x) = max(0, x·W1+b1)·W2+b2 # GELU: FFN(x) = GELU(x·W1+b1)·W2+b2

3️⃣ RMSNorm 归一化

使用 RMSNorm 替代 LayerNorm，去掉均值中心化，计算更快：

# LayerNorm: (x - mean) / sqrt(var + eps) # RMSNorm: x / sqrt(mean(x^2) + eps) ← 去掉减均值 RMSNorm(x) = x / RMS(x) * γ where RMS(x) = sqrt(mean(x^2) + eps)
特性 LLaMA-1 LLaMA-2 LLaMA-3
发布年份 2023.2 2023.7 2024.4
最大参数 65B 70B 405B
上下文长度 2K 4K 128K
词表大小 32K 32K 128K
GQA ❌ ✅ (70B) ✅

🔑 LLaMA-2 的关键升级：GQA

GQA (Grouped Query Attention)：多个 Query 头共享一组 K、V 头

  • MHA (Multi-Head): 8 Q头 + 8 K头 + 8 V头
  • MQA (Multi-Query): 8 Q头 + 1 K头 + 1 V头（省显存但效果差）
  • GQA (Grouped): 8 Q头 + 2 K头 + 2 V头（平衡方案）

好处：减少 KV Cache 显存占用，提升推理速度，效果接近 MHA。

🎯 面试常考点

  • LLaMA 和 GPT 的主要区别？
    → 位置编码（RoPE vs 可学习）、归一化（RMSNorm vs LayerNorm）、激活函数（SwiGLU vs GELU）
  • RoPE 为什么比绝对位置编码好？
    → RoPE 编码的是相对位置，具有更好的长度外推性
  • GQA 解决了什么问题？
    → 推理时 KV Cache 占用大，GQA 减少 KV 头数，节省显存

DeepSeek 系列 (DeepSeek-AI)

DeepSeek-AI 2023 - 至今 Decoder-only + MoE

📌 一句话定位

DeepSeek = 国产大模型的「技术先锋」
以极致的工程优化和创新的架构设计著称，DeepSeek-V2/V3 用极低的训练成本达到 GPT-4 级别性能，震惊业界。

🏗️ 核心创新

1️⃣ MLA (Multi-head Latent Attention) - DeepSeek-V2

DeepSeek-V2 的核心创新，颠覆传统 KV Cache：

# 传统 MHA: 每个头都有独立的 K, V # KV Cache 大小 = batch × num_heads × seq_len × head_dim # MLA: 压缩 K, V 到低维隐向量 (latent vector) c_k = W_DK · h # 压缩 key 到低维 c_v = W_DV · h # 压缩 value 到低维 # 推理时只存压缩后的 c_k, c_v，大幅减少显存 # 计算时再解压: k = W_UK · c_k

效果：KV Cache 减少 93.3%，推理速度大幅提升。

2️⃣ MoE 架构 (Mixture of Experts)

DeepSeek-V2/V3 采用 稀疏 MoE 架构：

  • 总参数量巨大（V3 达 671B），但每次只激活部分专家
  • 通过 Router 网络选择 Top-K 个专家处理每个 token
  • 激活参数量小（V3 每 token 只激活 37B），推理速度快
模型 总参数 激活参数 特色
DeepSeek-V2 236B 21B MLA + MoE
DeepSeek-V3 671B 37B MLA + MoE + MTP
DeepSeek-R1 671B 37B 推理模型，RL 训练

3️⃣ MTP (Multi-Token Prediction) - DeepSeek-V3

一次预测多个 token，加速训练收敛：

# 传统: 一次预测 1 个 token loss = CE(pred_t, target_t) # MTP: 同时预测未来 n 个 token loss = CE(pred_t, target_t) + CE(pred_t+1, target_t+1) + CE(pred_t+2, target_t+2) + ...

4️⃣ 训练成本优化

DeepSeek-V3 的训练成本约 557.6 万美元（对比 GPT-4 估计上亿美元），秘诀：

  • FP8 混合精度训练：首次在大规模模型上成功应用 FP8
  • DualPipe 并行策略：极致的流水线并行，减少气泡
  • 专家负载均衡：辅助损失确保各专家负载均衡

5️⃣ DeepSeek-R1 (推理模型)

对标 OpenAI o1 的推理模型，核心训练流程：

  1. 冷启动：用少量高质量 CoT 数据 SFT
  2. RL 训练：纯强化学习（GRPO 算法），无 SFT
  3. 拒绝采样：用 RL 后的模型生成高质量数据
  4. 全场景 SFT：用高质量数据训练，支持多种任务
  5. 最终 RL：对齐人类偏好

🎯 面试常考点

  • MLA 是什么？解决了什么问题？
    → Multi-head Latent Attention，通过压缩 KV 到低维隐向量，大幅减少 KV Cache 显存占用
  • MoE 的优势和劣势？
    → 优势：参数规模大、激活参数小、效果好；劣势：显存占用大（要存所有专家）、负载均衡难、工程复杂
  • DeepSeek 为什么训练成本低？
    → FP8 训练、极致并行优化、MoE 架构、高效的数据策略

高频面试 Q&A

为什么现在主流大模型都用 Decoder-only，而不是 Encoder-Decoder？
三点原因：
1. 生成任务主导：ChatGPT 带火了对话/生成任务，Decoder-only 更适合自回归生成
2. 效率更高：Decoder-only 架构简单，训练和推理更高效
3. Scaling Law：研究表明 Decoder-only 在参数规模扩大时表现更好

Encoder-Decoder 仍用于翻译、摘要等特定任务（如 T5、BART）。
RoPE 和绝对位置编码的本质区别是什么？
绝对位置编码：为每个位置分配唯一编码（如正弦编码、可学习嵌入），模型记住的是「第5个位置」

RoPE：通过旋转操作，让 attention score 只依赖于相对距离 (m-n)，模型学到的是「相隔3个位置」

关键优势：相对位置编码对未见过的长度泛化更好（外推性），因为相对距离是有限的（如最大512），而绝对位置是无限的。
MLA 和 GQA 都是优化 KV Cache，它们有什么区别？
GQA：减少 K、V 的头数（如8个头变2个），每个头维度不变
→ KV Cache 减少为 1/4

MLA：压缩 K、V 的维度到低维隐向量（如从 128 维压缩到 64 维）
→ KV Cache 减少为 1/2 或更少

DeepSeek 的做法：MLA + 解耦，KV Cache 减少 93%，效果几乎无损。
MoE 模型为什么「总参数量大，但推理速度快」？
关键：稀疏激活

· 总参数大：有 N 个专家，每个专家有 M 参数，总参数 = N×M
· 激活参数小：每个 token 只路由到 Top-K 个专家（如2个），实际计算参数 = K×M

举例：DeepSeek-V3 总参数 671B，每 token 只激活 37B，推理速度接近 37B 的 Dense 模型，但效果接近 671B。
Pre-Norm 和 Post-Norm 有什么区别？为什么现在都用 Pre-Norm？
Post-Norm（原始 Transformer）：
output = LayerNorm(x + Sublayer(x))
问题：梯度经过 LayerNorm 会缩小，深层网络梯度消失严重

Pre-Norm（现代大模型）：
output = x + Sublayer(LayerNorm(x))
优势：残差连接直接传递梯度，训练更稳定，支持更深网络（100+ 层）

代价：Pre-Norm 的表示能力略弱，但可以通过增加层数弥补。
SwiGLU 为什么比 ReLU/GELU 更好？
SwiGLU = 门控机制 + Swish 激活

公式：FFN(x) = (x·W₁ ⊗ Swish(x·W₂))·W₃

优势：
1. 门控机制：类似 LSTM 的门控，可以自适应地选择信息
2. Swish 激活：平滑非单调，比 ReLU 梯度更稳定
3. 表达能力：三个矩阵的乘积形式表达能力更强

代价：参数量略多（3个矩阵 vs 2个），通常把隐层维度从 4d 调整为 8d/3 来补偿。

🎯 面试核心要点

架构：都是 Decoder-only，Causal Mask，自回归生成
位置编码：GPT 可学习，LLaMA/DeepSeek 用 RoPE
归一化：都用 Pre-Norm，LLaMA/DeepSeek 用 RMSNorm
激活函数：LLaMA/DeepSeek 用 SwiGLU，GPT 用 GELU
注意力优化：GQA (LLaMA) → MLA (DeepSeek)
架构创新：DeepSeek 用 MoE + MLA 实现极致效率
