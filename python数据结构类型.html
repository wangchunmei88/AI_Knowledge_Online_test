📋 列表 (list) —— 有序、可变、随便装

特点：有序、可变（可增删改）、元素可重复、可混合类型。

# 创建列表
fruits = ['apple', 'banana', 'orange']
mixed = [1, 'hello', 3.14, True]

# 常用操作
fruits.append('grape') # 末尾添加
fruits.insert(1, 'pear') # 指定位置插入
fruits.remove('banana') # 删除元素
last = fruits.pop() # 弹出末尾元素
print(fruits[0]) # 索引访问 O(1)
✅ 适合：需要频繁增删、顺序遍历、修改元素
❌ 缺点：查找元素（in）和中间插入删除较慢（O(n)）
💡 常用方法：append(), extend(), insert(), pop(), remove(), sort(), reverse()

🔒 元组 (tuple) —— 不可变的“列表”

特点：有序、不可变（创建后不能增删改）、元素可重复。

# 创建元组
t = (1, 2, 3)
single = (5,) # 只有一个元素时逗号不能省
empty = ()

# 访问与列表一样
print(t[0]) # 1
# t[0] = 10 # 报错！不可修改
✅ 优点：不可变 → 可哈希 → 可作为字典的键，且比列表省内存。
✅ 适合：坐标点、数据库一行记录、常量配置、函数返回多个值。
🔍 解包技巧：a, b, c = (1,2,3) 或 a, *rest = (1,2,3,4)

📖 字典 (dict) —— 超快速的“键值对”

特点：键值对存储，键必须不可变（字符串、数字、元组），值可以是任意类型。无序（Python 3.7+ 保留插入顺序，但面试常说无序）。

# 创建字典
person = {'name': 'Alice', 'age': 25, 'city': 'Shanghai'}
# 或者 dict(name='Alice', age=25)

# 增改
person['job'] = 'Engineer' # 新增
person['age'] = 26 # 修改

# 删除
del person['city']
age = person.pop('age')

# 遍历
for k, v in person.items():
    print(k, v)
✅ 查找/插入/删除平均 O(1)，极快！
✅ 适合：缓存、配置映射、计数器、数据记录快速查询。
⚠️ 注意：字典的键必须是不可变类型。列表不能当键，但元组可以（如果元组内元素也是不可变）。

🌀 集合 (set) —— 去重 + 集合运算

特点：无序、元素唯一、可变。元素必须是可哈希的（类似字典的键）。

# 创建集合
s = {1, 2, 3, 3, 2} # 自动去重 → {1,2,3}
empty_set = set() # 注意 {} 是空字典
# 从列表去重
lst = [1,2,2,3,4,4]
unique = set(lst) # {1,2,3,4}

# 集合运算
a = {1,2,3}
b = {3,4,5}
print(a | b) # 并集 {1,2,3,4,5}
print(a & b) # 交集 {3}
print(a - b) # 差集 {1,2}
✅ 适合：去重、成员关系测试（in 很快）、数学集合运算。
⚠️ 没有顺序，不能通过下标访问。
💡 冻结集合 frozenset：不可变版本的集合，可以作为字典的键。

🎤 面试高频问题 & 回答思路

1. 列表和元组的区别？
答：列表可变，元组不可变；列表占用内存稍大，速度稍慢；元组可作字典的键，列表不行。另外元组比列表更轻量。
2. 字典底层怎么实现的？为什么查找快？
答：哈希表。通过哈希函数计算键的位置，冲突用开放寻址或链地址法解决。平均 O(1)，最坏 O(n)（但很少发生）。
3. 集合和列表的去重哪个快？
答：集合去重快（O(n)），因为底层哈希表；列表手动去重（if not in）是 O(n²)。大规模数据用 set(list)。
4. 元组真的完全不可变吗？
答：如果元组里包含可变对象（如列表），元组本身不可变（不能替换元素），但可以修改列表内部内容。例如 t = ([1,2], 3) 可以 t[0].append(4)。

💡 开发实用小技巧

🔹 列表推导式
[x*2 for x in range(10) if x%2==0]
🔸 字典推导式
{k: v for k,v in zip(keys, values)}
🔹 集合推导式
{x for x in 'abracadabra'} → 去重字母
🔸 统计元素频率
from collections import Counter
Counter([1,2,2,3])
# 合并两个字典 (Python 3.9+):
dict1 = {'a':1}; dict2 = {'b':2}
merged = dict1 | dict2 # {'a':1, 'b':2}

⚡ 性能小贴士

  • 查找元素：字典/集合 > 列表/元组（列表需遍历）
  • 频繁插入/删除末尾：列表快；中间插入删除：链表快（但Python标准库无内置链表，可用 collections.deque）。
  • 去重：用 set(list)，不要手动写循环。
  • 判断存在：if x in set/dict 极快；if x in list 慢。
