第一部分：Transformer 整体架构

8题
1
原始 Transformer（Attention is All You Need）的 Encoder 由哪些模块堆叠而成？单选
  • A. 自注意力层 + 前馈网络层，各自带残差连接和 LayerNorm
  • B. 自注意力层 + 交叉注意力层 + 前馈网络层
  • C. CNN 卷积层 + 自注意力层
  • D. RNN 循环层 + 注意力层
查看答案与解析
✅ 正确答案：A Encoder 每层由两个子层构成：
1. Multi-Head Self-Attention（多头自注意力）
2. Feed-Forward Network（FFN）（逐位置前馈网络）
每个子层都用 残差连接 包裹，再做 LayerNorm，即 output = LN(x + Sublayer(x))。
交叉注意力（Cross-Attention）只出现在 Decoder 中（用于关注 Encoder 输出），不在 Encoder 里。
2
Transformer Decoder 比 Encoder 多了哪个关键模块？单选
  • A. 位置编码层
  • B. 带掩码的自注意力层（Masked Self-Attention）
  • C. 交叉注意力层（Cross-Attention）和带掩码的自注意力层
  • D. 额外的 LayerNorm 层
查看答案与解析
✅ 正确答案：C Decoder 每层有 三个子层：
1. Masked Self-Attention：对已生成的 token 做自注意力，但用因果掩码（causal mask）遮住未来的 token，防止信息泄露
2. Cross-Attention：Q 来自 Decoder 自身，K/V 来自 Encoder 输出，让 Decoder 关注源序列信息
3. FFN：与 Encoder 相同的前馈网络

GPT 系列是 Decoder-Only，没有 Cross-Attention；BERT 是 Encoder-Only，没有 Decoder 部分。
3
Transformer 中 Scaled Dot-Product Attention 为什么要对点积结果除以 √d_k？单选
  • A. 为了加快计算速度
  • B. 防止点积值过大导致 softmax 进入梯度饱和区，造成梯度消失
  • C. 为了让注意力权重之和等于 1
  • D. 减少模型参数量
查看答案与解析
✅ 正确答案：B Q·Kᵀ 的结果方差约为 d_k（维度越大，点积值越大）。当点积值非常大时，softmax 的输出会趋向于 one-hot，梯度接近于 0（梯度消失），模型无法有效学习。
除以 √d_k 后，点积方差归一化为 1，softmax 输出更加平滑，梯度正常流动。
公式：Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V
4
在标准 Transformer 中，FFN（前馈网络）的隐层维度通常是模型维度 d_model 的多少倍？单选
  • A. 1 倍
  • B. 2 倍
  • C. 4 倍
  • D. 8 倍
查看答案与解析
✅ 正确答案：C 原始论文中 d_model=512，FFN 隐层维度=2048，即 4倍。这是一个经典的设计惯例，被大多数模型沿用。
计算：FFN(x) = max(0, xW₁ + b₁)W₂ + b₂，其中 W₁ ∈ ℝ^(d_model × d_ff)，d_ff = 4 × d_model。
现代大模型（如 LLaMA 使用 SwiGLU）为了弥补 SwiGLU 多一个矩阵的参数开销，通常将隐层缩为约 8/3 × d_model（≈2.67倍），以保持总参数量不变。
5
关于 Pre-Norm 和 Post-Norm，下列说法正确的有？多选
  • A. Post-Norm 是原始 Transformer 的设计，LayerNorm 在残差相加之后
  • B. Pre-Norm 将 LayerNorm 放在子层输入前，训练更稳定，现代大模型多采用
  • C. Pre-Norm 效果通常优于 Post-Norm，因为表达能力更强
  • D. 使用 Pre-Norm 通常可以省去 warmup 学习率调度
查看答案与解析
✅ 正确答案：A、B · A 正确：原始论文用 Post-Norm，即 output = LN(x + F(x))
· B 正确：Pre-Norm 即 output = x + F(LN(x))，梯度路径更干净，LLaMA/GPT-3 等均采用
· C 错误：Pre-Norm 训练更稳定但表达能力略弱（梯度不会通过 LN 后的 F），Post-Norm 理论上表达能力更强但难训练
· D 部分不准确：Pre-Norm 可以用更大的初始 lr，但通常仍需要 warmup，只是对 warmup 的依赖没那么强
6
Transformer 相比 RNN 的核心优势包括？多选
  • A. 并行计算：所有 token 可以同时处理，不依赖前一步的输出
  • B. 长距离依赖：任意两个 token 的注意力路径长度为 O(1)
  • C. 推理时对序列长度的计算复杂度更低（优于 RNN 的 O(n)）
  • D. 无梯度消失问题：残差连接使得梯度可以直接传到早期层
查看答案与解析
✅ 正确答案：A、B、D · A 正确：训练时整个序列并行计算，大幅提速
· B 正确：注意力可以直接连接任意两个位置，无需像 RNN 那样经过 n 步传递
· C 错误：Transformer 训练复杂度 O(n²d)，RNN 是 O(n)。Transformer 对长序列的计算代价更高，不是"更低"
· D 正确：残差连接 + LayerNorm 有效缓解梯度消失，使得深层模型可训练
7
请解释 Transformer 中残差连接（Residual Connection）的作用，以及为什么它对深层模型至关重要。简答
查看答案与解析
✅ 参考答案 残差连接的作用：
将子层的输入 x 直接加到输出上：output = x + F(x)，创建了一条"捷径（shortcut）"让梯度可以直接绕过子层反向传播。

为什么对深层模型至关重要：
1. 解决梯度消失：深层网络（如 96 层 GPT-3）没有残差连接时，梯度经过每一层都会乘以一个小于 1 的数，到底层时几乎为 0。残差连接保证了至少有 1 的梯度直接流过，梯度不会消失。
2. 保留原始信息：即使某个子层学到的变换很差，网络至少能"退化"成恒等映射 F(x)≈0，不会比原来更差（恒等学习更容易）。
3. 损失面更平滑：有残差的网络损失曲面更平滑，优化更容易收敛。
8
BERT（Encoder-Only）、GPT（Decoder-Only）、T5（Encoder-Decoder）三种架构各适合什么任务？请简要说明原因。简答
查看答案与解析
✅ 参考答案 BERT（Encoder-Only）：
· 适合：文本分类、命名实体识别（NER）、句子对任务（语义相似度、NLI）
· 原因：双向注意力，每个 token 能看到左右两侧的上下文，理解能力强；但不适合生成任务（无法做自回归生成）

GPT（Decoder-Only）：
· 适合：文本生成、对话、代码生成、In-Context Learning
· 原因：因果掩码的自回归结构天然适合逐 token 生成；大规模语言模型预训练后涌现出强大的 Few-Shot 能力

T5（Encoder-Decoder）：
· 适合：翻译、摘要、问答、任何 seq2seq 任务
· 原因：Encoder 理解输入，Decoder 通过 Cross-Attention 关注 Encoder 输出并生成目标序列，结构与任务天然匹配

Multi-Head Attention 中，每个头的维度 d_head 与总维度 d_model 和头数 h 的关系是？单选
  • A. d_head = d_model × h
  • B. d_head = d_model / h
  • C. d_head = d_model + h
  • D. d_head = √d_model
查看答案与解析
✅ 正确答案：B d_head = d_model / h。例如 d_model=512，h=8，则每个头 d_head=64。
这样设计使得多头注意力的总参数量和计算量与单头注意力相同（h 个头 × d_head² ≈ d_model²），但能从多个子空间捕获不同的语义信息。
每个头独立计算：head_i = Attention(QWᵢQ, KWᵢK, VWᵢV)，最后拼接再投影。
10
MHA、MQA、GQA 三种注意力机制，按 KV Cache 显存占用从大到小排列正确的是？单选
  • A. MQA > MHA > GQA
  • B. MHA > GQA > MQA
  • C. GQA > MHA > MQA
  • D. 三者相同
查看答案与解析
✅ 正确答案：B · MHA（Multi-Head Attention）：h 个 Q 头，h 个 K/V 头，KV Cache 最大
· GQA（Grouped Query Attention）：h 个 Q 头，g 组 K/V 头（g < h），每组多个 Q 共享一对 K/V，KV Cache 是 MHA 的 g/h
· MQA（Multi-Query Attention）：h 个 Q 头，只有 1 个 K/V 头，所有 Q 共享，KV Cache 最小（是 MHA 的 1/h）

LLaMA2/3、Mistral、Qwen2 等均采用 GQA，在效果和显存之间取得平衡。
11
因果注意力掩码（Causal Mask）在 Decoder 中的作用是？单选
  • A. 屏蔽 padding token，避免模型关注填充位置
  • B. 将注意力矩阵上三角部分设为 -∞，防止当前 token 看到未来 token
  • C. 减少注意力计算量，只计算对角线元素
  • D. 将注意力分数归一化到 [0,1] 区间
查看答案与解析
✅ 正确答案：B 自回归生成时，生成第 i 个 token 时只能看到位置 1 到 i 的 token，不能看位置 i+1 以后的（未来信息）。
实现方式：在 softmax 之前，将注意力分数矩阵的右上角（上三角）设为 -∞，经过 softmax 后变为 0，即完全忽略未来位置。
训练时可以并行处理整个序列，同时保证每个位置只看到历史信息，这是 Decoder 能高效训练的关键。
12
Flash Attention 相比标准注意力实现的核心创新是？单选
  • A. 将注意力计算复杂度从 O(n²) 降到 O(n log n)
  • B. 通过分块（Tiling）在 SRAM 中完成计算，避免将 O(n²) 的注意力矩阵写入 HBM
  • C. 使用近似方法跳过不重要的 token
  • D. 使用稀疏注意力减少计算量
查看答案与解析
✅ 正确答案：B GPU 内存层次：SRAM（片上缓存）速度极快但容量小；HBM（显存）容量大但 IO 慢。
标准注意力：QKᵀ（n×n矩阵）→ 写入 HBM → 读回做 softmax → 再写入 HBM，IO 开销是 O(n²)。
Flash Attention：将 Q/K/V 分块，每块在 SRAM 中计算，利用在线 softmax（online softmax）技术在不存储完整注意力矩阵的情况下得到正确结果，HBM IO 降至 O(n)。
结果：速度快 2-4×，显存从 O(n²) 降到 O(n)，计算量不变（仍 O(n²)）。
13
Sparse Attention（稀疏注意力）的主要思路是？单选
  • A. 每个 token 只与附近固定窗口内的 token 交互，或只与少数全局 token 交互
  • B. 随机丢弃 50% 的注意力权重
  • C. 只保留注意力矩阵的对角线元素
  • D. 把注意力矩阵量化为 INT8 精度
查看答案与解析
✅ 正确答案：A 稀疏注意力通过限制每个 token 的注意力范围来降低复杂度，常见模式：
· 局部窗口（Local Window）：每个 token 只关注周围 w 个 token，复杂度 O(nw)
· 全局 token（Global Token）：少数特殊 token（如 [CLS]）可以关注所有位置
· Sliding Window：Longformer/Mistral 的滑动窗口注意力
· Dilated（扩张）：类似空洞卷积，跳跃式关注
代表模型：Longformer、BigBird、Mistral（Sliding Window Attention）。将复杂度从 O(n²) 降到 O(n) 或 O(n√n)。
14
关于 KV Cache，下列说法正确的有？多选
  • A. KV Cache 只在推理（Decode）阶段使用，训练时不使用
  • B. 序列越长，KV Cache 占用的显存越多
  • C. GQA 可以减少 KV Cache 的显存占用
  • D. KV Cache 可以消除自回归生成的时间复杂度，使每步生成都是 O(1)
查看答案与解析
✅ 正确答案：A、B、C · A 正确：训练时所有 token 并行计算，不需要缓存；只有推理的逐步生成阶段才用 KV Cache
· B 正确：KV Cache 大小 ∝ 层数 × 头数 × 序列长度 × d_head × 2（K和V），序列越长占用越大
· C 正确：GQA 减少了 K/V 头数，KV Cache 相应减小
· D 错误：有了 KV Cache，每步生成时注意力的复杂度是 O(n)（新 token 的 Q 与所有历史 K 做点积），不是 O(1)，随序列增长而增长
15
注意力机制中 Q、K、V 分别代表什么含义？下列类比正确的有？多选
  • A. Q（Query）类似于搜索引擎里的查询词，代表"我想找什么"
  • B. K（Key）类似于数据库索引，代表"我能匹配什么"
  • C. V（Value）是实际取出的内容，代表"找到后返回什么"
  • D. Q、K、V 都是对输入 x 做不同线性变换得到的，参数不共享
查看答案与解析
✅ 正确答案：A、B、C、D 全部正确！这是理解注意力机制最直观的类比：
· Q（Query）：当前 token 的"提问"，要去和所有 K 比对
· K（Key）：每个 token 提供的"标签"，用于和 Q 计算相关性得分
· V（Value）：每个 token 实际携带的信息，按注意力权重加权求和
· 三者都是 Q=xWQ, K=xWK, V=xWV，用不同权重矩阵对同一输入 x 做线性投影，参数完全独立
16
PagedAttention（vLLM 提出）解决了什么问题？下列描述正确的有？多选
  • A. 将 KV Cache 类比操作系统的内存分页，非连续存储，减少显存碎片
  • B. 允许多个请求共享相同前缀的 KV Cache（Prefix Caching）
  • C. 将注意力计算的精度降低到 FP8 以节省计算资源
  • D. 显著提升推理吞吐量（throughput），尤其在高并发场景下
查看答案与解析
✅ 正确答案：A、B、D · A 正确：传统 KV Cache 需要连续显存，容易碎片化，浪费空间。PagedAttention 用"块"（page）存储，类似 OS 虚拟内存，空间利用率大幅提升
· B 正确：共享同一 System Prompt 的多个请求可以共用那部分的 KV Cache，减少重复计算（Prefix Caching 是 vLLM 基于 PagedAttention 实现的优化）
· C 错误：PagedAttention 不涉及精度降低，这是量化的事情
· D 正确：PagedAttention 使 vLLM 的吞吐量比 HuggingFace 原生实现高出约 24 倍
17
请详细解释自注意力（Self-Attention）和交叉注意力（Cross-Attention）的区别，并说明各自的应用场景。简答
查看答案与解析
✅ 参考答案 自注意力（Self-Attention）：
· Q、K、V 全部来自同一序列：Q=xWQ, K=xWK, V=xWV（x 是同一个输入）
· 作用：让序列中每个 token 关注序列中其他 token，建模序列内部的依赖关系
· 应用：Encoder 的自注意力（理解输入），Decoder 的掩码自注意力（理解已生成序列）

交叉注意力（Cross-Attention）：
· Q 来自一个序列，K 和 V 来自另一个序列：Q=x₁WQ, K=x₂WK, V=x₂WV
· 作用：让一个序列的每个位置关注另一个序列的信息，建模跨序列的依赖关系
· 应用：Encoder-Decoder 架构中，Decoder 用 Cross-Attention 关注 Encoder 输出（如翻译时关注源语言）；扩散模型中图像特征关注文本特征（如 Stable Diffusion 的文本条件注入）
18
Speculative Decoding（投机解码）的原理是什么？它如何在不改变输出分布的前提下加速生成？简答
查看答案与解析
✅ 参考答案 核心思路：用小模型（Draft Model）快速预测多个 token，再用大模型（Target Model）并行验证。

步骤：
1. Draft 小模型自回归生成 k 个候选 token（如 k=4）
2. Target 大模型将这 k 个 token 一次性并行输入（prefill 模式），计算每个位置的概率分布
3. 按照拒绝采样（rejection sampling）逐一验证：若 Draft 的 token 被 Target 接受则保留，否则从 Target 的分布重新采样并截断后续 Draft token
4. 平均每次大模型 forward 可接受多个 token（加速比取决于小模型和大模型的分布相似度）

关键保证：输出分布与直接用 Target 模型生成完全相同（数学上等价），没有质量损失，只有速度提升（通常 2-3×）。
代表实现：vLLM 的 Speculative Decoding，以及 Medusa（同一模型多头预测）。

19
原始 Transformer 使用正弦/余弦位置编码，其公式为 PE(pos,2i)=sin(pos/10000^(2i/d))。以下关于它的说法错误的是？单选
  • A. 不需要额外学习参数，是固定编码
  • B. 不同维度使用不同频率，低维度变化快，高维度变化慢
  • C. 理论上可以泛化到比训练序列更长的位置
  • D. 直接将位置编码向量加到词嵌入向量上
查看答案与解析
✅ 正确答案：B（错误说法） B 是错误的。正确说法是：低维度（i 小）频率高，变化快；高维度（i 大）频率低，变化慢。因为分母 10000^(2i/d) 随 i 增大而增大，使得 pos/10000^(2i/d) 随 i 增大而减小，正弦函数变化更缓慢。这类似于二进制数中低位变化快、高位变化慢。
A、C、D 都是正确的：固定编码无需学习；数学上对任意长度都有定义（理论可外推）；实现上直接相加（词嵌入 + 位置编码）。
20
RoPE（旋转位置编码）为什么被称为"旋转"位置编码？单选
  • A. 它将 token 在空间中旋转排列
  • B. 它通过旋转矩阵将位置信息注入 Q 和 K，使点积结果只依赖相对位置
  • C. 它对注意力权重矩阵进行旋转变换
  • D. 它是循环（circular）位置编码的缩写
查看答案与解析
✅ 正确答案：B RoPE 的核心思想：将 Q 和 K 中的每两个维度视为一个复数平面上的向量，根据位置 m/n 旋转对应角度。
关键性质：⟨RoPE(q,m), RoPE(k,n)⟩ = f(q, k, m-n)
即两个向量的内积只依赖于它们的相对位置 (m-n)，而不依赖于绝对位置，这使得 RoPE 天然具有相对位置感知能力和更好的外推性。
LLaMA1/2/3、Qwen、Mistral 等主流大模型全部使用 RoPE。
21
如何让基于 RoPE 的模型支持超过训练长度的序列（长上下文外推）？单选
  • A. 直接增大 batch size
  • B. 使用 Position Interpolation（位置插值）或 YaRN 等方法对 RoPE 的旋转角度进行缩放
  • C. 增加 Transformer 层数
  • D. 将词表大小扩大一倍
查看答案与解析
✅ 正确答案：B 若模型训练时最大长度为 4096，推理时遇到 8192 长度的序列，位置编码会进入未见过的区间，效果下降。
常用外推方案：
· Position Interpolation（PI）：将位置索引线性压缩到训练范围内（如将 0-8191 映射到 0-4095），再少量微调
· YaRN：更精细地对不同频率的 RoPE 维度进行不同程度的缩放，效果优于简单 PI
· LongRoPE：非均匀缩放，进一步提升外推能力
这些方案都只需少量长文本微调（甚至不微调）即可将上下文窗口扩展数倍。

ALiBi（Attention with Linear Biases）位置编码的特点是？单选
  • A. 将位置向量加到 Q 和 K 上
  • B. 不修改 embedding，直接在注意力分数上加一个与距离成比例的负偏置
  • C. 使用正弦函数编码绝对位置
  • D. 将位置信息编码到 V 而不是 Q/K
查看答案与解析
✅ 正确答案：B ALiBi 的公式：注意力分数 = QKᵀ/√d + m·(-(|i-j|))
其中 m 是每个头的斜率（不同头不同斜率，无需学习），|i-j| 是两 token 的距离。
特点：
· 不需要位置嵌入向量，实现极简单
· 对更远的 token 施加更大的负惩罚，自然建模"距离越远越不相关"
· 超长序列外推性较好（距离惩罚是连续的）
· 代表模型：BLOOM（176B）使用 ALiBi

关于不同位置编码方案的对比，下列说法正确的有？多选
  • A. 可学习绝对位置编码（如 BERT）超出训练长度后完全失效
  • B. RoPE 通过相对位置编码机制，外推性优于可学习绝对位置编码
  • C. ALiBi 直接在注意力分数上操作，不占用 embedding 的表示维度
  • D. RoPE 和 ALiBi 都不需要额外的可学习参数
查看答案与解析
✅ 正确答案：A、B、C、D 全部正确：
· A：BERT 的位置 embedding 只训练了 512 个，超出后没有对应参数，模型完全不知道怎么处理
· B：RoPE 基于相对位置，理论上对任意长度的相对距离都有定义，只是超出训练范围后分布不同，需要插值/微调辅助
· C：ALiBi 不修改 x 的表示，只在注意力计算过程中加 bias，干净简洁
· D：RoPE 的旋转角度由公式确定（10000^(2i/d)），无可学习参数；ALiBi 的斜率也是预设的，无可学习参数
24
为什么 Transformer 必须引入位置编码？如果不加位置编码会发生什么？简答
查看答案与解析
✅ 参考答案 为什么需要位置编码：
Transformer 的注意力机制本质上是对集合（set）操作——对所有位置的 token 同时计算注意力，整个过程是位置无关的（permutation equivariant）。即把序列打乱顺序，注意力的计算结果只是打乱顺序，不会产生"理解"序列顺序的能力。

不加位置编码会怎样：
· "我爱你"和"你爱我"对模型来说完全一样（打乱 token 顺序后输入相同）
· 模型无法理解语序、句法结构，文本的语序信息完全丢失
· 模型退化为一个"词袋"（bag-of-words）模型，无法处理自然语言

解决方案：通过位置编码（绝对/相对）将位置信息注入，让模型知道每个 token 在序列中的位置，从而理解语序和距离关系。

Transformer 训练时常用的 Label Smoothing（标签平滑）技术的作用是？单选
  • A. 加快模型收敛速度
  • B. 防止模型对真实标签过度自信，提升泛化能力，缓解过拟合
  • C. 减少模型参数量
  • D. 平衡类别不均衡问题

Transformer 训练常用 Warmup + Cosine Decay 学习率调度，Warmup 阶段的目的是？单选
  • A. 让模型快速收敛到最优解
  • B. 训练初期参数随机，梯度方差大，小学习率热身避免参数更新过猛导致训练崩溃
  • C. 节省计算资源
  • D. 防止模型在训练初期过拟合
查看答案与解析
✅ 正确答案：B 训练最初几步，模型参数是随机初始化的，梯度估计不准确（方差很大），如果学习率太大，参数更新幅度过大，会使模型进入很差的区域，甚至 loss 爆炸（NaN）。
Warmup：从很小的学习率（如 1e-7）线性增加到目标学习率（如 1e-4），经过几百到几千步后再按 Cosine 曲线衰减。
这给了模型足够的时间"找好方向"再大步走，是大模型训练的标配策略。

混合精度训练（Mixed Precision Training）通常使用 FP16 和 FP32，下列说法正确的是？单选
  • A. 前向传播和反向传播都用 FP32，只有参数存储用 FP16
  • B. 前向传播和反向传播用 FP16，保留 FP32 的主参数副本用于更新，并使用 Loss Scaling
  • C. 所有计算全部用 FP16，不需要 FP32
  • D. 只有 Embedding 层用 FP16，其余用 FP32
查看答案与解析
✅ 正确答案：B 标准 AMP（自动混合精度）流程：
1. 保留一份 FP32 的主参数（master weights）副本
2. 前向传播：参数转 FP16 计算，激活值用 FP16 存储（省显存）
3. 反向传播：梯度用 FP16 计算，再转 FP32 更新主参数
4. Loss Scaling：FP16 最小值约 6e-8，梯度可能下溢为 0，将 loss 放大一个系数（如 1024），反向传播后再还原，防止梯度下溢
好处：显存节约约 50%，计算速度提升（V100/A100 的 Tensor Core 对 FP16 有专项优化）。

大模型训练中常见的并行策略，下列说法正确的有？多选
  • A. 数据并行（Data Parallel）：每个 GPU 有完整的模型副本，不同 batch 并行计算
  • B. 张量并行（Tensor Parallel）：将单个权重矩阵按列/行切分到多个 GPU
  • C. 流水线并行（Pipeline Parallel）：将模型的不同层分配到不同 GPU
  • D. ZeRO（零冗余优化）：将优化器状态、梯度、参数分片到不同 GPU，减少每卡显存
查看答案与解析
✅ 正确答案：A、B、C、D 全部正确！四种主要并行策略：
· 数据并行（DP）：最简单，PyTorch DDP；缺点是每卡需存完整模型
· 张量并行（TP）：Megatron-LM；将矩阵列切分，需要 all-reduce 通信，适合单机多卡
· 流水线并行（PP）：前几层在 GPU0，后几层在 GPU1...；缺点是有 bubble（空泡），GPipe/PipeDream 缓解
· ZeRO（DeepSpeed）：Stage1 切分优化器状态，Stage2 再切梯度，Stage3 再切参数；Stage3 相当于完全切碎，1000 张卡可训万亿参数模型
实际大模型训练通常组合 3D 并行（DP + TP + PP）+ ZeRO。

  • A. 目的是减少激活值的显存占用
  • B. 代价是在反向传播时需要重新计算部分前向激活值，增加约 33% 的计算量
  • C. 可以将激活值显存从 O(n) 降低到 O(√n)（n 为层数）

什么是 Dropout？Transformer 中在哪些地方使用？训练和推理时有何区别？简答
查看答案与解析
✅ 参考答案 Dropout 是什么：训练时以概率 p 随机将神经元输出置为 0，防止神经元之间形成共适应（co-adaptation），起正则化作用，等效于训练了大量共享权重的子网络的集成。

Transformer 中使用位置：
· 注意力权重上：对 softmax 后的注意力权重做 Dropout（attn_dropout）
· FFN 中间层激活后
· 残差连接前（sub-layer output dropout）
· 词嵌入 + 位置编码叠加后
原始论文使用 p=0.1。大模型（如 GPT-3）有时使用 p=0 不做 Dropout，依赖大数据防止过拟合。

训练 vs 推理：
· 训练时：以概率 p 随机丢弃，剩余激活值乘以 1/(1-p) 保持期望不变（inverted dropout）
· 推理时：Dropout 完全关闭，所有神经元都参与计算，不做任何丢弃

量化（Quantization）中 GPTQ 和 AWQ 的共同核心思路是？单选

  • B. 都需要在量化后进行完整的模型微调

查看答案与解析

  • B. Beam size=1 时退化为贪心解码

  • D. Beam size 越大，生成质量一定越高

什么是模型蒸馏（Knowledge Distillation）？在 Transformer 模型压缩中如何应用？简答
查看答案与解析
✅ 参考答案 知识蒸馏：用已训练好的大模型（Teacher）指导小模型（Student）训练，让 Student 学习 Teacher 的"软标签"（概率分布）而非硬标签（one-hot）。

核心损失函数：
L = α · L_CE(student_logits, hard_labels) + (1-α) · L_KD(student_logits/T, teacher_logits/T)
其中 L_KD 是 KL 散度，T 是蒸馏温度（通常 T=4），用软化的概率分布传递 Teacher 的"暗知识"。

在 Transformer 中的应用：
· DistilBERT：BERT 蒸馏到 6 层，保留 97% 性能，速度快 60%
· 中间层蒸馏：不仅对齐最终输出，还对齐中间层的注意力矩阵和隐状态（TinyBERT）
· LLM 蒸馏：大模型生成的回答作为 Student 的训练数据（如 Alpaca 用 GPT-4 的输出训练小模型，这也称为"数据蒸馏"）

请解释什么是"涌现能力"（Emergent Abilities）？它在大模型中是如何体现的？为什么在小模型上不出现？简答
查看答案与解析
✅ 参考答案 涌现能力：指在小模型上几乎不存在（随机水平）、但当模型规模超过某个临界值后突然出现的能力，不是随规模线性增长的，而是"质变"。

典型例子：
· 思维链推理（Chain-of-Thought）：约 100B 参数以上模型才能做复杂多步数学推理
· 上下文学习（In-Context Learning）：几个例子就能完成新任务，小模型几乎做不到
· 指令遵循：理解并遵从复杂指令的能力

· 也有观点认为"涌现"是评测指标选择的产物（用连续指标可能是平滑增长的）
